HTML

  • The html is the framework for the webpage - it describes the structure, labelling different elements
  • Semantic HTML elements indicate the role of their content and give structure to the content of the webpage
  • The head element contains information about the webpage, but is not visible to the user
  • The meta element goes in the head and provides information to the browser about how the page will be rendered
  • The head element can also contain other attributes, such as the language of the site, who made the site, and the title, which is important for SEO
  • The body element contains the content that is visible to the user

CSS

  • Cascading Style Sheets (CSS) is the language used to style an HTML document and describes how HTML elements should be displayed
  • There are multiple ways to introduce CSS - inline, internal and external
  • 'Inline style' changes the attributes of a single element
  • An 'internal style sheet' is embedded in the head, and contains rules for that webpage
  • 'External css style sheet' is the most common way to style a website - it keeps design rules together and makes changes easier. It is referenced with a link element in the HTML head
  • There are a wide variety of CSS selectors available, allowing precise selection and styling of elements, some selectors include: type, class, ID and attribute selectors
  • In a CSS sheet, styling elements applies to the entire page. Styling class attributes applies to specific parts of the page and can be done by creating a selector
  • Everything in CSS has a box around it, and understanding these boxes is key to being able to create layouts with CSS, or to align items with other items.
  • A margin indicates how much space we want around the outside of an element
  • Padding indicates how much space we want around the content inside an element

Git

  • Working with git allows different developers to work on the same project concurrently - it allows version control and avoids conflicts or sync errors
  • It is essential to work on new branches when making any changes to a project, they can then be checked and merged with the main branch without over-writing work
  • Frequently sync the local version with the remote version in GitHub so a functional, updated version of the project can be shared and accessed by all team members
  • git status: checks what branch we are currently on
  • git checkout -b new-branch-name: creates a new branch and switches to it. Use conventions when naming, for example feature/detail
  • git add -A: adds modifications in the current working branch to the staging area; the -A flag indicates we want to add all changes
  • git commit -m "description of commit": creates a commit that captures the currently staged changes, ready to be pushed
  • N.B. The convention is to write the description of a commit as an imperitive
  • git pull origin main: use this before commiting, to check the local branch is in sync with the base branch - do this everytime you start to work
  • git push origin new-branch-name

JavaScript

  • Javascript adds interactivity to a static HTML/CSS page
  • It is a scripting language that enables you to create dynamically updating content, control multimedia, animate images, and many many other things
  • Basic JavaScript features include variables, strings and data types
  • Control flow is the order in which a computer executes code in a JavaScript file
  • Understanding control flow allows conditional logic to be added to the Prework Study Guide
  • Arrays can be used to store data types in JavaScript, they can be thought of as how to store a group of related data in a variable
  • A for loop can be used to operate on each element in an array
  • Do not add your .js link to the header, it needs to load after the html so place it at the bottom of the body, before closing the tag
  • JavaScript can store useful values inside variables
  • It can perform operations on pieces of text (known as "strings")
  • It can run a code in response to certain events
  • Order really matters: the objects referenced have to exist before doing anything to them
  • Variables are containers that store values, or put another way, it is a named container that allows us to store data in our code
  • One way to declare a variable is to use the var keyword followed by a unique name
  • The equals sign is the assignment variable to assign specific data
  • Quotation marks denote a string
  • An if statement is a way of introducing decision-making into our code. For more than one condition, use else if
  • Comments with line breaks have an asterisk and a forward slash; without line breaks two forward slashes can be used
  • At the most basic level, a function is simply a set of instructions that tells the computer how to perform a certain task, they execute when called by name, unlike for loops and conditional statements, which automatically execute
